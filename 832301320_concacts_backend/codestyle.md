# Code Style Guide

This code style guide is based on:
- JavaScript: Airbnb JavaScript Style Guide (https://airbnb.io/javascript/)
- Python: PEP 8 -- Style Guide for Python Code (https://www.python.org/dev/peps/pep-0008/)
- HTML/CSS: Google HTML/CSS Style Guide (https://google.github.io/styleguide/htmlcssguide.html)

## JavaScript/Vue.js

1. Use 2 spaces for indentation
2. Use camelCase for variable and function names
3. Use PascalCase for Vue components
4. Always use semicolons at the end of statements
5. Use single quotes for strings
6. Add spaces around operators (except for unary operators)
7. Use === instead of == for comparisons

## Python/Flask

1. Use 4 spaces for indentation
2. Use snake_case for variable, function, and module names
3. Use PascalCase for class names
4. Limit lines to 79 characters
5. Add a blank line between functions and classes
6. Import statements should be on separate lines
7. Use meaningful variable and function names

## HTML

1. Use lowercase for all HTML elements and attributes
2. Use double quotes for attribute values
3. Always close HTML tags
4. Use proper indentation to show hierarchy
5. Include a doctype declaration at the beginning
6. Use semantic HTML elements when possible

## CSS

1. Use lowercase for all CSS selectors and properties
2. Use hyphens for multi-word class names
3. Use 2 spaces for indentation
4. Include a space after the colon in property declarations
5. Use shorthand properties where possible
6. Group related styles together
